## Anti-Patterns Verdict
LLM assessment: does NOT read as AI slop. The fire-assay concept (bone-ash paper, muted gold bead, inscriptional Hedvig Letters Serif, numbered assay-steps, vault tree) is a specific POV, not a template. None of the absolute bans present: no side-stripe accents, no gradient text, no glassmorphism, no hero-metric template, no identical card grid, no modal.
Deterministic scan: detect.mjs not bundled in this skill install; deterministic scan unavailable. Compensated with browser inspection across all pages, both themes, desktop + mobile.
Browser evidence: verified index/canon/office in light + dark, desktop (1440) + mobile (450). No console errors. Scroll-spy, theme persistence, drop caps, and responsive folds all confirmed working.

## Bugs found and fixed this pass
- Vault file-tree rendered as a collapsed run-on blob (a div with newlines but no `white-space: pre`). Fixed; renders as an aligned tree.
- Drop caps never appeared (the kicker `<p>` was intercepting `:first-of-type`). Fixed when the step header collapsed; caps now render.
- Three mismatched left edges (rail 626 / masthead 786 / body 846). Unified to one column.
- Canon top rail overflowed and hid "Categories" behind a horizontal scroll. Resolved by the sidebar refactor.
- Footer wrap (1080) wider than content column; aligned to 740.
- Categories table cramped at ~625px; stacking breakpoint raised to 720px.

## What's Working
- Navigation: persistent left sidebar with scroll-spy is always-visible and scannable; folds to a wrapped top list on mobile.
- Typography: clear three-tier hierarchy (inscriptional display, Spectral body, mono data); drop caps and fineness numbers add character without noise.
- Restraint: bone-ash paper + ink + a single muted gold accent reads calm and confident; one callout style, no decoration for its own sake.

## Priority Issues (minor)
- [P3] No client-side search/filter. Fine at this size; revisit if the docs grow.
- [P3] No `og:image` for social cards (omitted rather than ship a broken ref).
- [P2] Domain `cupel.trading` sits oddly against the "never places trades" stance; `cupel.gold`/`cupel.ink` are more on-brand.
